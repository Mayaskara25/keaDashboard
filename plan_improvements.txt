====================================================================================
QUESTION CLASSIFIER TOOL
DESIGN IMPROVEMENTS
Version 1.1
(Addendum to QUESTION_CLASSIFIER_PLAN.md)
====================================================================================

PURPOSE
-------

These are architectural improvements to the existing Question Classifier Tool.

The overall pipeline remains unchanged.

The goal is to

- improve solution quality
- reduce Gemini API cost
- make future migrations easier
- improve maintainability
- make the ingestion pipeline reproducible


====================================================================================
CHANGE 1
VERIFY OFFICIAL ANSWERS INSTEAD OF SOLVING BLINDLY
====================================================================================

Current Pipeline

Question
↓

Gemini solves question

↓

Returns answer


Recommended Pipeline

Question

↓

Check if answer key exists

↓

If YES

Prompt Gemini:

"The official answer key says Option C.

Verify whether this answer is correct.

If correct,
explain why.

If incorrect,
provide the correct answer,
explain the mistake in the official key,
and continue using the corrected answer."

↓

Store verified answer.


Benefits

• Detect OCR mistakes.

• Detect incorrect answer keys.

• Better explanations because the model reasons about an expected answer.

• Increases trust in imported content.


====================================================================================
CHANGE 2
SEPARATE SOLVING FROM CLASSIFICATION
====================================================================================

Current Pipeline

Question

↓

Gemini

↓

Solution

↓

Metadata


Recommended Pipeline

Question

↓

Step 1
Gemini solves the question

↓

Intermediate JSON

{

    correctOption,

    explanation,

    reasoning

}

↓

Step 2

Gemini classifies

↓

Metadata JSON

{

    conceptId,

    skill,

    difficulty,

    misconceptions,

    bloomsLevel,

    expectedTime,

    reasoningSteps,

    knowledgeElements

}

↓

Merge outputs

↓

Final Question JSON


Reason

Solving chemistry problems and generating educational metadata are different
tasks.

Keeping them separate

• improves consistency

• makes prompt tuning easier

• allows replacing one stage without affecting the other


====================================================================================
CHANGE 3
STORE LLM CONFIDENCE
====================================================================================

Every generated solution should include

confidence

Example

{

    "correctOption":"B",

    "confidence":"HIGH"

}

or

{

    "confidence":0.96

}

Suggested categories

HIGH

MEDIUM

LOW


Usage

HIGH

↓

Automatic approval possible

MEDIUM

↓

Quick manual review

LOW

↓

Mandatory manual verification


This allows maintainers to prioritize which generated solutions require human
attention.


====================================================================================
CHANGE 4
REWRITE EXPLANATIONS FOR TEACHING QUALITY
====================================================================================

Current

Question

↓

Gemini

↓

Explanation

↓

Store


Recommended

Question

↓

Gemini

↓

Initial Explanation

↓

Gemini Prompt

"Is this explanation suitable for a KCET/JEE coaching student?

If not,

rewrite it so that

• it teaches the concept

• explains why wrong options are wrong

• highlights common mistakes

Keep it concise."

↓

Store final explanation


Purpose

The objective is not merely to produce the correct answer.

The explanation should help students learn.

Teachers will also read these explanations.

High-quality explanations increase the educational value of the platform.


====================================================================================
CHANGE 5
DO NOT SOLVE QUESTIONS THAT ALREADY HAVE SOLUTIONS
====================================================================================

Current

Every question

↓

Gemini Solve

↓

Gemini Classify


Recommended

Question

↓

Does the PDF already contain a solution?

YES

↓

Extract existing solution

↓

Skip solving

↓

Only perform classification

↓

Store

NO

↓

Gemini Solve

↓

Gemini Classify

↓

Store


Benefits

• Saves approximately 30–40% of API usage.

• Preserves official explanations where available.

• Speeds up ingestion.


====================================================================================
CHANGE 6
STORE LLM METADATA
====================================================================================

Every generated question should include metadata describing how it was
generated.

Example

{

    "llm_metadata":{

        "model":"gemini-2.5-flash",

        "generated_at":"2026-06-28T10:15:00Z",

        "processing_version":1,

        "confidence":"HIGH"

    }

}

IMPORTANT

This metadata DOES NOT belong in the production Questions table.

Instead,

keep it

• in the intermediate JSON

or

• in a staging database

before importing into production.


Benefits

• Easy regeneration when prompts improve.

• Track which model generated each solution.

• Track processing pipeline versions.

• Reprocess only low-confidence questions.

• Future-proof migration when switching to newer models.


====================================================================================
CHANGE 7
RECOMMENDED FINAL INGESTION PIPELINE
====================================================================================

Teacher uploads PDF

↓

Extract text / OCR

↓

Split into individual questions

↓

Extract answer key (if available)

↓

IF explanation already exists

    Extract explanation

ELSE

    Gemini Solve

↓

Gemini verifies official answer (if answer key exists)

↓

Gemini Classify

↓

(Optional)
Gemini rewrites explanation for teaching quality

↓

Validate JSON against schema

↓

Duplicate detection

↓

Teacher review (only if confidence is LOW or validation warnings exist)

↓

Import into Question Database


====================================================================================
FUTURE OPTIMIZATION
====================================================================================

Many concepts repeat across thousands of questions.

Example

Aldol Reaction

appears hundreds of times.

Instead of generating a complete explanation for every question,

build a Concept Explanation Library.

Concept

↓

Master Explanation

↓

Question-specific reasoning

↓

Final explanation

Advantages

• Consistent teaching style

• Lower token usage

• Easier maintenance

• Better educational quality


====================================================================================
DESIGN PRINCIPLE
====================================================================================

The ingestion pipeline should prioritize

1. Correctness
2. Reproducibility
3. Maintainability
4. Low API cost
5. Educational quality

Runtime AI remains zero.

All AI computation happens only once during content ingestion.
